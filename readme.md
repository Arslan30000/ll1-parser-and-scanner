# CustomLang Compiler Frontend

This repository contains the frontend components of a custom compiler built from scratch in Java. It is developed as part of the Compiler Construction course (CS4031) and is divided into two major phases: Lexical Analysis and Syntax Analysis.

## Phase 1: Lexical Analyzer (Scanner)
The lexical analyzer reads `.lang` source files and converts the raw text into a stream of structured tokens.

### Key Features:
* **Dual Implementation:** Built using both a Manual Procedural DFA and an automated generator (JFlex).
* **Token Support:** Strictly recognizes 7 specific token categories: Identifiers, Integer Literals, Float Literals (including scientific notation), Boolean Literals, Arithmetic Operators, Punctuators, and Single-Line Comments.
* **Symbol Table:** Tracks variable names and their frequencies.
* **Error Handling:** Catches and logs lexical errors (e.g., malformed floats, invalid characters, exceeding length limits) without crashing.

## Phase 2: Syntax Analyzer (LL(1) Parser)
The syntax analyzer validates the sequence of tokens generated by the scanner against a Context-Free Grammar (CFG).

### Key Features:
* **Grammar Transformation:** Automatically eliminates Left Recursion and performs Left Factoring (In Progress).
* **Automated Set Computation:** Dynamically computes `FIRST` and `FOLLOW` sets for any given grammar.
* **Predictive Parsing:** Constructs a 2D LL(1) Parsing Table.
* **Stack-Based Execution:** Parses input token streams using a pushdown automaton (Stack).
* **Panic Mode Error Recovery:** Gracefully handles syntax errors by skipping offending tokens or popping mismatched terminals from the stack to resume parsing.
